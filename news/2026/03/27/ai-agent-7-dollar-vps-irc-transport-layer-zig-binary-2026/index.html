Developer Builds Fully Functional AI Agent on a $7 Per Month VPS Using IRC as Transport Layer

Developer Builds Fully Functional AI Agent on a $7 Per Month VPS Using IRC as Transport Layer

What Happened

A developer known as George Larson has captured the attention of the Hacker News community by building and deploying a fully functional AI agent system running on a $7 per month virtual private server, using the decades-old Internet Relay Chat protocol as its communication transport layer. The project, documented in a detailed technical blog post on March 23, 2026, demonstrates that sophisticated AI agent architectures don't require expensive cloud infrastructure or proprietary platforms.

The system consists of two agents running on separate machines. The public-facing agent, called "nullclaw," is a remarkably compact 678 kilobyte Zig binary that uses approximately 1 megabyte of RAM and connects to an Ergo IRC server. Visitors interact with it through a gamja web client embedded in Larson's website. The private agent, "ironclaw," handles email and scheduling tasks and is accessible only over Tailscale VPN using Google's Agent-to-Agent (A2A) protocol.

The project employs a tiered inference strategy: Anthropic's Haiku 4.5 handles routine conversations at sub-second latency and minimal cost, while Sonnet 4.6 is invoked only when tool use is required. A hard spending cap of $2 per day ensures the system remains sustainable. The architecture has been running continuously for over a week with total AI costs under $10.

Background and Context

The AI agent ecosystem has been dominated by well-funded startups and major technology companies deploying complex, expensive infrastructure. Platforms like AutoGPT, CrewAI, and LangChain have created frameworks that, while powerful, often require significant compute resources and cloud services that can cost hundreds or thousands of dollars per month for production deployments.

Larson's project represents a counter-narrative to this trend. By choosing Zig — a systems programming language designed for performance and small binary sizes — and IRC — one of the oldest and most battle-tested communication protocols on the internet — he has demonstrated that the essential capabilities of an AI agent system can be achieved with minimal resources. The choice of IRC is particularly inspired: the protocol's simplicity, reliability, and low overhead make it an ideal transport layer for text-based AI interactions.

The use of Google's A2A protocol for inter-agent communication adds a modern twist to the retro-inspired architecture. A2A, which Google released as an open standard for agent interoperability, allows the two agents to delegate tasks to each other using a standardized protocol — the private agent effectively "borrows" the public agent's conversational interface when needed.

Why This Matters

This project challenges the prevailing assumption that AI agents require enterprise-grade infrastructure. The demonstration that a useful AI assistant can run on commodity hardware with a $7 monthly budget has implications for individual developers, small businesses, and organizations in developing countries that want to deploy AI capabilities without the cost of major cloud platforms.

The technical choices also highlight the importance of efficiency in AI deployment. While many organizations run AI workloads on powerful GPU instances, Larson's agents perform their computation through API calls to hosted AI models, keeping the local footprint tiny. This architecture — lightweight local orchestration with remote inference — could become a template for cost-effective AI agent deployment across many use cases.

For businesses that manage their own IT infrastructure with genuine Windows 11 key workstations and basic server setups, this project demonstrates that AI agents don't require a separate expensive technology stack. A small VPS or even a repurposed desktop machine can serve as the foundation for useful AI automation that handles email, scheduling, and customer interactions.

Industry Impact

The project has sparked significant discussion about the democratization of AI agent technology. While companies like Microsoft, Google, and Anthropic invest billions in AI infrastructure, the actual cost of deploying a useful AI agent — when built with efficient tools and sensible architecture — is remarkably low. This disconnect suggests that much of the enterprise AI market's pricing reflects convenience and brand premium rather than fundamental infrastructure costs.

The IRC transport layer choice is generating particular interest among developers who are frustrated with the complexity of modern communication platforms. IRC's simplicity — it's essentially a text protocol with channels — means there are no API rate limits to worry about, no OAuth flows to maintain, and no platform risk from a company changing its terms of service. For AI agents that primarily deal in text, this simplicity is a feature rather than a limitation.

For the enterprise productivity software market, the project illustrates how AI agents can be built to integrate with existing infrastructure rather than requiring new platforms. An organization already using affordable Microsoft Office licence tools could add AI agent capabilities through lightweight integrations rather than purchasing an enterprise AI platform — a potentially significant shift in how businesses approach AI adoption.

The tiered inference model — using cheap models for routine tasks and more expensive models only when needed — is a pattern that more organizations should adopt. Many enterprise AI deployments use the most capable (and expensive) model for every interaction, when a simpler model would suffice for the majority of use cases.

Expert Perspective

What makes this project technically impressive is not any single innovation but the holistic efficiency of the design. Every choice — Zig for the binary, IRC for transport, Tailscale for secure networking, A2A for inter-agent communication, tiered inference for cost management — optimizes for a different dimension of the system. The result is an architecture that achieves 90 percent of what enterprise AI agent platforms deliver at perhaps 1 percent of the cost.

The 678 KB binary size deserves special attention. In an era where simple web applications routinely ship hundreds of megabytes of dependencies, building a functional AI agent orchestrator in under a megabyte is a remarkable feat of engineering discipline. It suggests that the AI agent ecosystem would benefit from more attention to efficiency and less from frameworks that prioritize developer convenience over runtime performance.

What This Means for Businesses

Small businesses and individual professionals should take note: the barrier to deploying a personal AI agent is far lower than the enterprise AI market suggests. A $7 VPS, API access to a language model, and some development effort can produce a system that handles email triage, scheduling, and basic customer interactions — capabilities that enterprise platforms charge thousands of dollars per month to provide.

For technology leaders evaluating AI agent platforms, this project provides a useful benchmark. If a solo developer can build a functional agent system for under $10 per month, what exactly are enterprise vendors providing for $10,000 per month? The answer may be support, compliance, and integration — valuable things, but the underlying infrastructure cost is clearly far lower than most enterprise pricing suggests.

Looking Ahead

Larson has indicated plans to open-source the agent framework, which could enable a wave of low-cost AI agent deployments. The project has already inspired several community members to build similar systems using different programming languages and transport protocols. As AI model costs continue to decline and open-source models improve, the economics of self-hosted AI agents will only become more attractive — potentially creating a grassroots alternative to the enterprise AI agent platforms that currently dominate the market.
